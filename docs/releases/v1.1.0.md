# Kanji Widget v1.1.0

Release date: 2026-03-11

## Highlights

- Refined the Kanji Detail layout with separate Onyomi and Kunyomi sections
- Replaced the old today-summary block with explicit Today total, Valid opens, and This kanji metrics
- Added hero metadata on the detail screen for stroke count, grade, and frequency when available
- Added a Next random action directly on the detail screen so review can continue without returning to the launcher

## Improvements

- Reused one shared detail-intent pipeline across widget, launcher, stats, and detail navigation
- Improved detail-screen fallback behavior for missing metadata and unavailable random catalogs
- Added a dedicated Kanji Detail design document and aligned overview documentation with the shipped behavior

## Notes

- Existing local study history and cached kanji data remain on-device
- Release builds continue to use the existing GitHub Actions signing workflow

# Kanji Widget v1.3.1

Release date: 2026-03-12

## Highlights

- Fixed Kanji Detail audio controls so unusable reading placeholders no longer appear as playable targets
- Kept compound-example rows visible when a supported word is available but its reading is missing
- Preserved the lightweight detail flow while making audio availability clearer on-device

## Improvements

- Disabled audio controls more consistently when Japanese `TextToSpeech` is not ready on the device
- Reused the shared unavailable-reading placeholder for compound rows that do not have a reading value
- Updated the Kanji Detail design notes to match the shipped playback and compound-row behavior

## Notes

- This release does not change the production package name or the existing study data format
- Existing stroke-order playback, today stats, and next-random navigation remain unchanged
- The debug build still installs separately from the production app through its debug application ID

# Releases

This folder stores curated release notes for published releases and release-note drafts already prepared on `master`.

Rule:
- Create one file per release tag.
- If release notes are prepared before the tag is created, keep the filename aligned with the intended version and update or publish it when the tag is cut.
- Keep the content short and user-facing.
- Match the release tag name, for example `v1.1.0.md`.

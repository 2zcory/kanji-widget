# Detail Design

This folder stores detailed design documents for individual features.

Rule:
- Create one document per feature.
- Use clear file names such as `widget-random-mode.md` or `stroke-order-screen.md`.

Suggested sections for each document:
- Purpose
- Scope
- UI behavior
- Data flow
- Storage or API impact
- Edge cases
- Testing notes

Diagram rule:
- Follow `docs/diagram-standards.md`.
- New durable feature docs should usually include one Mermaid `sequenceDiagram` or feature-level `flowchart` for the main interaction path.
